## Task

Classify a short WAV recording of a spoken Hindi digit into one of **11 classes**:

| Class | Hindi | Romanized |
|-------|-------|-----------|
| 0 | аӨ¶аҘӮаӨЁаҘҚаӨҜ | shunya |
| 1 | аӨҸаӨ• | ek |
| 2 | аӨҰаҘӢ | do |
| 3 | аӨӨаҘҖаӨЁ | teen |
| 4 | аӨҡаӨҫаӨ° | chaar |
| 5 | аӨӘаӨҫаӨҒаӨҡ | paanch |
| 6 | аӨӣаӨ№ | chhah |
| 7 | аӨёаӨҫаӨӨ | saat |
| 8 | аӨҶаӨ  | aath |
| 9 | аӨЁаҘҢ | nau |
| 10 | вҖ” | **Unknown** (non-digit / OOV) |

## Dataset

**SMAI Dataset (wav)** вҖ” WAV recordings organized into per-class folders (0вҖ“10). Not included in this repo; place it at the root before running anything.

```
SMAI Dataset (wav)/
в”ңв”Җв”Җ 0/    вҶҗ аӨ¶аҘӮаӨЁаҘҚаӨҜ recordings
в”ңв”Җв”Җ 1/    вҶҗ аӨҸаӨ•   recordings
в”ңв”Җв”Җ ...
в””в”Җв”Җ 10/   вҶҗ unknown / non-digit speech
```

### Augmentation

Run `augment_dataset.py` **once before training**. It:

- Creates an 80/20 stratified train/test split on the original files вҶ’ saves `dataset_split.json`
- Generates 5 augmented variants per training clip using: time stretching (0.85вҖ“1.15Г—), pitch shifting (Вұ3 semitones), additive Gaussian noise, volume perturbation (0.5вҖ“1.5Г—), and speed perturbation
- Saves augmented WAVs to `SMAI Dataset (wav) - Augmented/`

```bash
python augment_dataset.py
```

> The test set uses **original files only** to ensure fair evaluation.

## Approaches

### Approach 1 вҖ” Fine-tuned Whisper (tiny)

`openai/whisper-tiny` with the **encoder frozen** and the decoder fine-tuned to transcribe Hindi digit words. At inference, the transcription is matched against a keyword dictionary (Hindi + English + romanized forms). Any transcription that doesn't match a known digit вҶ’ class 10 (Unknown).

| | |
|---|---|
| Base model | `openai/whisper-tiny` |
| Frozen | Encoder |
| Optimizer | AdamW, lr=5e-5, cosine schedule |
| Batch size | 8 |
| Early stopping | patience=2 |

### Approach 2 вҖ” Wav2Vec2 + MLP Classifier

Frozen `facebook/wav2vec2-base` encoder produces 768-dim mean-pooled embeddings. A small 2-layer MLP head is trained on top. Embeddings are extracted **once and cached** вҖ” only the head trains, making epochs very fast.

```
Audio вҶ’ [Wav2Vec2 encoder, frozen] вҶ’ mean-pool вҶ’ 768-dim
      вҶ’ Linear(768вҶ’256) вҶ’ ReLU вҶ’ Dropout(0.4) вҶ’ Linear(256вҶ’11)
```

| | |
|---|---|
| Base model | `facebook/wav2vec2-base` |
| Frozen | Entire encoder |
| Optimizer | Adam, lr=1e-3, cosine schedule |
| Batch size | 64 |
| Early stopping | patience=20 |

### Approach 3 вҖ” MFCC + 2D CNN

MFCC + О” + О”О” features stacked into a 3Г—40Г—100 tensor, fed to a lightweight 2D ConvNet trained **from scratch** вҖ” no pretrained model required.

```
Audio вҶ’ MFCC (40 coeff) + О” + О”О” вҶ’ (3, 40, 100) tensor
      вҶ’ 3Г— [Conv2d вҶ’ BN вҶ’ ReLU вҶ’ MaxPool вҶ’ Dropout]
      вҶ’ AdaptiveAvgPool вҶ’ Linear(2048вҶ’256) вҶ’ Linear(256вҶ’11)
```

| | |
|---|---|
| Optimizer | Adam, lr=1e-3, weight decay=1e-4, cosine schedule |
| Batch size | 16 |
| Early stopping | patience=3 |

## Training

All three training pipelines are in the unified notebook:

```bash
jupyter notebook all_approaches.ipynb
```

Run cells top to bottom. Each approach:
1. Loads the augmented training data
2. Trains with early stopping (best val accuracy checkpoint saved)
3. Evaluates on the original test set
4. Outputs a results plot (loss curve, accuracy curve, confusion matrix)

Trained model weights are saved locally but not tracked in this repo (see `.gitignore`).

## Live Inference

Hold **Enter** to record, release to get a prediction. Press **Esc** to quit.

```bash

python inference/live_inference1_whisper.py
python inference/live_inference2_wav2vec2.py
python inference/live_inference3_mfcc_cnn.py
```

Approaches 2 and 3 display a top-3 confidence breakdown. Approach 1 shows the raw Whisper transcription and the matched keyword.

**macOS:** System Settings вҶ’ Privacy & Security вҶ’ Microphone вҶ’ enable for Terminal.
